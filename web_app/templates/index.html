Esterilización de disco destino. ADVERTENCIA: Destruirá todos los datos en el disco seleccionado.

Sin dispositivo seleccionado
Estándar HDD/USB: Buffer auto-ajustado (4–16 MB) para máximo throughput secuencial. Buffer grande = menos overhead I/O.

2. Extraer Disco (dc3dd)

Adquisición física bit a bit con cálculo de hash al vuelo y Data Carving. Cumple norma ISO/IEC 27037:2012.

Sin dispositivo seleccionado
Checklist ISO/IEC 27037
  • 1. Bloqueador de Escritura Activado (Solo Lectura)
  • 2. Almacenamiento Seguro Destino Montado
  • 3. Actas y Cadena de Custodia Listas para Emisión
3. Creador USB RAM

Prepara una memoria USB con herramientas estáticas para volcado de RAM en el equipo sospechoso.

PELIGRO DE PÉRDIDA DE DATOS: El USB seleccionado será FORMATEADO permanentemente (exFAT). Todos sus datos actuales se destruirán. Asegúrese de seleccionar el dispositivo correcto.

4. Análisis RAM Volátil

Análisis forense de memoria con Volatility3. Detecta el SO automáticamente y ejecuta todos los plugins correspondientes.

Windows Linux macOS ISO/IEC 27037 · NIST SP 800-86
Sin archivo seleccionado — Explorar USB, /media o /mnt
Plugins Volatility3 (por OS detectado):
pslist pstree cmdline netscan malfind dlllist hashdump bash_history lsof
Progreso: 0% Iniciando análisis...
5. Extracción Móvil Lógica

Adquisición lógica forense de dispositivos móviles via ADB (Android) o libimobiledevice (iOS). Cumplimiento ISO/IEC 27037 y NIST SP 800-101.

Android (ADB) iOS (libimobiledevice) ISO/IEC 27037 · NIST SP 800-101
Requisito forense: El dispositivo debe estar desbloqueado con Depuración USB activa (Android) o "Confiar en este equipo" aceptado (iOS). Esta extracción es de tipo Lógica (Nivel 2 - NIST).
Artefactos que se extraerán:
📱 Info dispositivo (IMEI) 💬 SMS / MMS 📞 Historial llamadas 👤 Contactos 📦 Apps instaladas 🖼️ DCIM / Fotos ⬇️ Downloads 💚 WhatsApp Media
📱 Info dispositivo (UDID/IMEI) 💾 Backup lógico completo 📦 Apps instaladas
